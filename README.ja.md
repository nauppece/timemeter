# タイムメーター (obsidian-timemeter)

> For English, see [README.md](README.md).

最前面アプリを自動追跡して `TimeMeter/YYYY-MM-DD.md` に Markdown で記録する Obsidian プラグインです。「タップして一言メモ」で “そのセッションで何をしていたか” を残せるようにするのが主眼で、自動追跡そのものは主役ではありません。

- 自動追跡（最前面アプリ・任意でウィンドウタイトル・AFK 検知）ができるのは **macOS デスクトップ版のみ**。
- 記録できるのは **Obsidian を開いている間だけ** です。閉じている/スリープ中の操作は追跡されません。
- モバイル（iOS/iPadOS/Android）では自動追跡は行われませんが、データの閲覧（サイドバー・埋め込み）と手動ログの追加はできます。UI は日英切替対応（既定は英語）。

> **⚠️ システムコマンドの実行について（重要）**
> 自動追跡のため、macOS デスクトップ版では `child_process` 経由で macOS 標準コマンドを実行します：`osascript`（最前面アプリ名・ウィンドウタイトル）と `ioreg`（無操作＝AFK 秒数）。実行は `Platform.isDesktopApp` が真のときだけで、モバイルでは一切呼ばれません（`child_process` も読み込みません）。`execFile`（シェル非経由）で静的引数のみ・AppleScript も静的文字列で変数展開なし（インジェクション経路なし）、システムを**読むだけ**で、**外部送信は一切しません**（すべて Vault 内の Markdown に保存）。

## インストール

### 1. コミュニティプラグイン経由（審査通過後）

設定 → コミュニティプラグイン → 閲覧 で **「TimeMeter」** を検索 → インストール → 有効化。

### 2. BRAT 経由（ベータ／今すぐ）

1. コミュニティプラグインから **BRAT**（Obsidian42 - BRAT）を導入・有効化。
2. BRAT の "Add Beta plugin" に `nauppece/timemeter` を追加。
3. 設定 → コミュニティプラグインで **TimeMeter** を有効化。

### 3. 手動（リリース）

[Releases](https://github.com/nauppece/timemeter/releases) の最新版から `main.js` / `manifest.json` / `styles.css` を `<vault>/.obsidian/plugins/timemeter/` に置いて有効化。

```sh
# ソースからビルドして自分の Vault に配置
git clone https://github.com/nauppece/timemeter.git
cd timemeter
npm install
export TIMEMETER_PLUGIN_DIR="/path/to/YourVault/.obsidian/plugins/timemeter"
npm run deploy
```

> macOS で自動追跡を使うには、初回に **オートメーション**と**アクセシビリティ**の許可が必要です（後述）。許可するまではサイドバーに権限エラーが出ますが、閲覧・手動ログは動きます。

## できること / できないこと

### デスクトップ (macOS)

- アプリの自動追跡（ポーリング間隔は設定可）。既定では**デスクトップに開いている全アプリを並行して記録**（同時作業も残せる。`captureAllApps` で最前面のみに切替可）。
- **既定では AFK（無操作）を無視して途切れず記録**。設定「離席（AFK）検知」を ON にすると、無操作の時間を止めずに**時系列で「離席」として薄色表示**（合計には含む・しきい値既定10分）。
- ウィンドウタイトルも記録（**最前面アプリのみ**。ホバー詳細と日報の LLM 補完に使用）。
- サイドバー: 今日／日別／月（ヒートマップ・日クリックで日別へ）。
- 合計バーのアプリをクリック → モーダルで **追記先を選択**（先頭=デイリー／その下に最近使ったファイル）し内容を入力 → 選んだファイルに `- アプリ: 内容` を追記。
- 時系列レーンの空セグメントをクリック → 同じピッカー。既定は **「この記録の説明」**（データファイルの説明列）。
- クイックコマンド: 今のセッションにメモ／手動ログ／ラップ。
- ステータスバー、`timemeter` 埋め込み、日報下書き挿入、LLM 用プロンプトのコピー。

### モバイル

- サイドバーでの閲覧・`timemeter` 埋め込み・手動ログ追加。
- **自動追跡（最前面アプリ検知・AFK 検知）はできません**（「今のセッションにメモ」「ラップ」は非表示）。

### 共通の制約

- 自動追跡はデスクトップ版 Obsidian が起動している間だけ。閉じた/スリープ中は記録されません。
- 追跡対象は「最前面アプリが何か」だけで、アプリ内の詳細な操作は見ていません。

## 権限（macOS）

1. **オートメーション許可**（最前面アプリ名に必須）：初回追跡時にダイアログ。出ない/拒否した場合は `システム設定 → プライバシーとセキュリティ → オートメーション → Obsidian → System Events` を ON。
2. **アクセシビリティ許可**（ウィンドウタイトルに必須）：`システム設定 → プライバシーとセキュリティ → アクセシビリティ → Obsidian` を ON。無くてもアプリ名の記録は続きます（タイトルが空になるだけ）。

権限が無い/拒否されても例外は握りつぶして `null` を返すため、クラッシュやエラー表示はありません。AFK 検知（`ioreg -c IOHIDSystem`）は追加権限不要です。

## コマンド一覧

コマンドパレット（`Cmd/Ctrl+P`）から。パレット上の完全な id は `timemeter:<id>`。

| コマンド | id | 実行可否 |
|---|---|---|
| パネルを開く | `open-view` | デスクトップ・モバイル |
| 今すぐ集計 | `aggregate-now` | 両方（モバイルは実質ノーオペ） |
| 今のセッションにメモ | `note-current` | **デスクトップのみ**・追跡中のアプリがある時だけ表示 |
| 手動ログを追加 | `manual-log` | 両方 |
| ラップ（ここから別作業） | `lap` | **デスクトップのみ** |
| デイリーに今日のタイムメーターを挿入 | `insert-daily-embed` | 両方 |
| 日報の下書きをデイリーに挿入 | `insert-nippou-draft` | 両方 |
| LLM 用プロンプトをコピー | `copy-claude-prompt` | 両方 |

デフォルトのホットキーは設定していません（他と衝突し得るため）。設定 → ホットキーで任意に割り当ててください。リボン（砂時計）からもパネルを開けます。

## データ形式

記録先は `<データフォルダ>/YYYY-MM-DD.md`（既定 `TimeMeter`、設定で変更可）。

### frontmatter

```yaml
---
date: 2026-07-09
total_min: 125
totals:
  "Obsidian": 80
  "✍️ Manual": 45
---
```

`date` / `total_min` / `totals` の3キーのみ管理（他キーは保持）。`totals` はアプリ別合計分。

### セッション表

本文中のマーカー間に Markdown テーブルで書かれます。

```
<!-- timemeter:sessions:start -->
| 開始 | 終了 | 時間 | アプリ | タイトル | 説明 | 離席 |
|------|------|------|--------|----------|------|------|
| 09:00 | 10:15 | 1h 15m | Obsidian | note.md — Vault | 資料整理 |  |
<!-- timemeter:sessions:end -->
```

列は **開始 / 終了 / 時間 / アプリ / タイトル / 説明 / 離席** の7列（手動ログはアプリ列 `✍️ Manual` の行で区別。旧 `✍️ 手動` も認識し、次回書き込みで `✍️ Manual` に統一）。見出しの文言はプラグインの言語設定に追従します（日本語なら上記、English にすると `Start / End / Dur / …`）。既存ファイルは次回書き込み時に現在の言語の見出しへ書き換わります。**離席**列は AFK 検知 ON のとき無操作区間に `1`（表示の色分け用・合計には含む）。旧6列ファイルもそのまま読め、次回書き込みで7列化されます。

- マーカー外の本文は一切変更しません。セル内の `|`→`\|`、改行→`<br>` にエスケープ。タイトルに管理マーカーが混入しても無害化します。
- **再集計してもデータは失われません**（`開始HH:MM｜アプリ` キーでマージ。説明は新しい方が空なら既存を残す。手動ログ等の片側のみの行も保持）。

## LLM 連携

「LLM 用プロンプトをコピー」で以下のようなプロンプトをコピーします（API キー不要。当日のファイルと一緒に Claude Code などへ）。

```
`TimeMeter/2026-07-09.md` の説明が空のセッションについて、時間帯とアプリ・ウィンドウタイトルから内容を推測して説明列を埋めて。既にある説明は変更しないで。
```

## 設定

| 設定 | キー | 既定値 | 範囲 |
|---|---|---|---|
| 言語 | `lang` | `en` | English / 日本語 |
| ポーリング間隔（秒） | `pollIntervalSec` | 10 | 5〜60（5刻み） |
| 離席（AFK）検知 | `afkDetect` | false | OFF＝無操作でも記録／ON＝離席を色分け |
| 離席とみなす時間（分） | `afkThresholdSec` | 600 | 1〜30分（`afkDetect` ON 時のみ） |
| 結合ギャップ（分） | `mergeGapMin` | 3 | 1〜10 |
| 開いている全アプリを記録 | `captureAllApps` | true | ON/OFF（OFFで最前面のみ） |
| データフォルダ | `dataFolder` | `TimeMeter` | 任意（空欄で既定に戻る） |
| デイリーの見出し | `dailyHeading` | `## TimeMeter` | 追記/埋め込み/日報の挿入先（空欄で末尾） |
| ステータスバー表示 | `showStatusBar` | true | ON/OFF |
| 起動時にサイドバーを開く | `showSidebarOnStart` | true | ON/OFF |
| アプリ別ルール（表示/除外） | `apps[アプリ名]` | 未観測は `{ hidden: false }` | アプリごとに表示/除外 |

### 開いている全アプリを記録

毎ポーリングで最前面だけでなく開いている全アプリを記録します。**合計は実時間を超えることがあります**（同時作業ぶん重複）。離席中は記録されません。タイトルは最前面のみ。不要なアプリは除外してください。OFF で最前面のみ。

### 言語（日英切替）

パネル UI・通知・設定・ステータスバーは即反映。**コマンド名・リボン・プラグイン一覧名は Obsidian 仕様で登録時言語に固定**のため、変更にはリロードが必要です。

### アプリの除外

「表示」を OFF にすると時系列・合計・月・日報から**完全に除外**（データは残るのでいつでも復帰可）。除外中は設定上部にまとまります。サイドバーの ⋯／右クリックからも可。

### 時系列のズーム・スクロール

上部の `− / {倍率} / +` で 1x〜4x。はみ出しは横スクロール（トラックパッド・Shift+ホイール・ドラッグ）。中央時刻を保持、アプリ名列は左固定、ズームはセッション内のみ保持（今日/日別で独立）。

## 埋め込み

````
```timemeter
date: today
```
````

`date` は `today` または `YYYY-MM-DD`（省略・空・不正は `today`）。

## 開発

```sh
npm run dev     # esbuild watch
npm run build   # tsc --noEmit + esbuild production
npm test        # vitest run（84テスト）
npm run deploy  # build 後、TIMEMETER_PLUGIN_DIR のフォルダへ配置
```

ビルド成果物（`main.js` / `manifest.json` / `styles.css`）のみを vault に配置します。`deploy.sh` は `TIMEMETER_PLUGIN_DIR`（または gitignore した `deploy.local.sh`）から宛先を読むため、どの環境でも動きます。リリースは GitHub Actions がビルド＋証明（provenance attestation）付きで発行します。`githooks/pre-commit`（`npm install` で有効化）が個人パス・API キーの混入をブロックします。
